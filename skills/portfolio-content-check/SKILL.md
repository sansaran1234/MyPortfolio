# Portfolio Content Check

## Purpose
Review portfolio content changes for credibility, clarity, and presentation accuracy.

## Use When
- project details change
- links, labels, location, availability, or experience copy changes
- case study or CTA text changes

## Inputs
- changed content files
- visible labels and metadata in UI components

## Review Flow
1. Identify all changed user-facing copy and metadata.
2. Check for clarity and professional tone.
3. Check for inconsistencies across sections.
4. Flag broken trust signals such as mismatched labels or outdated references.

## Checklist
- project names and descriptions are consistent
- links and labels match intended destinations
- personal metadata reads clearly and professionally
- status or availability text does not conflict with other sections
- copy supports the premium technical tone of the site

## Completion Criteria
- content issues are called out explicitly
- trust-impacting inconsistencies are prioritized
